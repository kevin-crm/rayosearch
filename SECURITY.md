# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AISearchGen, **please do not open a public GitHub issue.**

Instead, report it privately by opening a [GitHub Security Advisory](../../security/advisories/new) or emailing the maintainer at [kevin@kevincrm.com](mailto:kevin@kevincrm.com).

Please include:
- A description of the vulnerability
- Steps to reproduce it
- Any potential impact you've identified
- Your suggested fix (optional but appreciated)

You can expect an acknowledgment within 48 hours and a resolution timeline within 7 days for critical issues.

## Scope

The following are in scope:
- Authentication and session handling
- API key generation and validation
- SQL injection, XSS, CSRF
- Any vulnerability that could expose user data

## Out of Scope

- Bugs that require physical access to the server
- Social engineering attacks
- Issues in third-party dependencies (report those upstream)

Thank you for helping keep AISearchGen secure.
